    Building an Application with Django

    Overview

    The technology and the proficiency required to build custom-fit solutions that reach a high level of expertise is quite challenging. Tools like web frameworks make the web application development painless.

    Let us take a look at what is a Web Framework? During end of 90's and beginning of 20's developers had to write boiler plate code to build web applications. Each web applications were developed from scratch. This approach was not scalable when it came to non trivial web applications. Many questions arised:

    • How to reduce time taken to market, if you do every development from scratch?
    • How complex web pages like interacting with multiple databses can be developed?
    • How to reuse similar code across multiple applications?
    • How can developer concentrate on business logic rather than writing bolier plate code?
    Web application frameworks are evolved to solve these problems. A web application framework is a programming infrastructure for your applications. Where the developer concentrate on busines logic than the boiler plate code and helps to reduce the time taken to reach market. Allows reusing of features and also helps to write clean and maintainable code. Django is one such framework with many features to make developer's life easy.

    What is Django?

    Django is quite a famous and well-featured server side framework, written in Python. Django enables hasty development of protected as well as sustainable websites. Django encourages you to build complete, adaptable, versatile, secure and lightweight softwares.

    In view of the quantity of prominent sites that make use of Django, the number of individuals consistently adding to the codebase of this framework, and the count of persons giving both free and paid for support options, we can imagine Django's popularity.

    Django Framework

    Django is a free and open source framework that deals with a great part of the issue of web development, such that now you can concentrate completely on creating your application without bothering about remodeling a pre-existing solution. Thus, it relieves a huge portion of the overhead involved in creating a new site.

    Django follows the "Batteries included" concept making it a complete product in itself. With this framework, one can build nearly any type of website. Moreover, Django is capable of working with various categories of client-side frameworks.

    Django can run your app on different platforms which makes it highly portable.

    First django project

    In Djangot we often come across Projects and Apps. A project refers to the entire application and all its parts. An app refers to a submodule of the project.

    Project Location

    Create an project folder in a prefered location of your computer. 

                  
$ mkdir django_tutorial

    Virtual Environment

    Before installing Django, it's prescribed to introduce a self-contained directory tree, a Virtualenv that establishes new disconnected environments which separate your Python records for every project you create.
    Go inside the project location (cd django_tutorial) to create the virtual environment using following command. 

              
In MacOS command-line
django_tutorial$ python3 -m venv tutorialenv 

In Windows command-line
C:\Users\Name\django_tutorial>python -m venv tutorialenv

    Install Django

    Now, since we created a virtual environment, next step will be to activate it using the below command. 

              
In MacOS command-line
$source tutorialenv/bin/activate

In Windows Command-line
C:\Users\Name>tutorialenv/Scripts/activate
    As soon as you find the prompt in your console having (tutorialenv) as prefix, this implies that your virtual env has been initiated.

    Finally, we are now ready to Install Django. There are 3 methods to install Django, the best being the one using pip command which gives us the latest stable version of Django. So in this tutorial, we suggest the same pip tool for installation. 
            
In MacOS Command-line
(tutorialenv) ~django_tutorial$pip3 install django

In Window
command -line
(tutorialenv) C:\Users\Name>tutorialenv/Scripts>pip install django

    Start the project

    Let’s begin to build our First project and see how to start it. The initial step is to start a new Django project. Essentially, this implies that we'll run a few scripts given by Django that will make a frame of a Django project. This is only a collection of directories and records that we will be utilizing later on. The names of certain files are significant for Django. Note an important suggestion at this point, you ought not rename or change the location of such files for a Django project. In order to start the project, follow these steps - 

                
For MacOS In command-line
(tutorialenv) ~$ django-admin startproject tutorial_site .
(Note the period at the end is mandatory.) 

For Windows In command-line
(tutorialenv) C:\Users\Name>django-admin startproject tutorial_site
    Next stage is the setting up of Database. In this project we will be using the default database, sqlite3. So, we just need to do migrations using the next command on the console - 
                
In command-line
(tutorialenv) ~$ python manage.py migrate
    We now try to start our first project, just type in the following command on the console - 
            
In command-line
(tutorialenv) ~$ python manage.py runserver
    Finally we have built our first website which is running on localhost.
    In order to view this, just navigate to the following URL on your local web browser. http://127.0.0.1:8000
    You get to see your website running !!!

    Django Models

    Django web applications access and manage data from databse and write to database through Python objects known as models. Whenever you've picked what database you need for your app(Which is supported by Django), you are free from the burden of writing complex queries, as Django will manage all communications with the database on your behalf.

    A model is a class that enables to interact with a table in our DB, and where each property of the class is a field of the table.

    In Django model is the built-in component that Django uses to generate tables and fields. Models are characterized in the app/models.py file. so we open this file residing within our app’s folder (app - give here the name of your own application) edit the code in IDE to look like:

    The final step here is to include our newly created model Book to our database. But first we will have to ensure Django knows the changes we did to our new model Book. In order to do this we need to make migrations using following command. 
                  
In command-line
(tutorialenv) ~$ python manage.py makemigrations tutorial-site
    Django is now ready with the migration file which will now be added to our database once we type this command python manage.py migrate tutorial-site on the console. 
                              
In command-line
(tutorialenv) ~$ python manage.py migrate tutorial-site
    Finally, with these steps we have added our self-created model Book to the database.

    Django URLs

    The urls.py file specifies which view is called for a mentioned URL. Whenever a user generates a request for any page of your site, Django controller looks through the urls.py file, to find the respective view thus returning the proper HTML response. Open the app/urls.py file located within our app’s folder (app - give here the name of your own application) and edit the code in IDE to look as : 

      
# book/urls.py

1 from django.urls import path
2 from . import views
3 urlpatterns = [(path ‘ ‘ , views.book_list, name=’book_list’),
     # you can add more paths here
]
    We are importing Django’s function path and all of the views from your app. Line 3 is important as it defines a mapping between URLs and views. In the sample, we assign a view named as book_list to the root URL.

    Django ORM

    Object-relational mapper(ORM) is used to transfer data between model object and relational database. The ORM automates this data transfer, and the developer need not write any SQL. ORM helps to write error-free free code and helps to increase productivity.

    Django Admin

    A web-based administrative backend is a standard component of most current sites. This interface, called as admin, permits confided site heads to build, alter and distribute content, manage the site clients and perform other managerial functions. Django accompanies an underlying administrator interface. With Django's administrator you can confirm clients, show and handle forms, and approve input; all autonomously. The Django admin application uses your models in order to automatically generate a site area which can be utilized to create, display, edit and remove records. The Django project suggests to use it only for use internally within your organization, since this interface provides a lot of irrelevant details to the clients (which are best if hidden). [1] All the configuration required to include the admin application in our website was done automatically when we created our First Django Project(which was discussed earlier in the tutorial). Now we are left with registering the models we created in the previous stage, so as to add them to the admin application. In order to achieve this, let’s open the app/admin.py file located within our app’s folder (app - give here the name of your own application) and edit the code in IDE to look as : 
      
# app/admin.py
1 from django.contrib import admin
2 
3 #Register your model Book
4
5 from .models import Book
6 
7 admin.site.register(Book)
    Using this sample code, you can register a model (or multiple models) with the admin site. We can also try creating a superuser, by following - 
      
For MacOS: In command-line
(tutorialenv) ~$ python manage.py createsuperuser

For Windows: In command-line
(tutorialenv) C:\Users\Name> python manage.py createsuperuser
    Now, a new superuser has been added to the database, which can be viewed if you restart the development server.

    Django vs Flask

    SNO

    Comparison

    Django

    Flask

    1

    Type of Framework

    Full stack web framework - can create full-scale powerful web apps in Django.

    WSGI framework- used for lightweight web applications where high speed is required

    2

    Flexibility

    Feature-Packed – Doesn’t allow changes to its modules.

    Full flexibility - can be extended with various development tools and libraries

    3

    ORM

    Built-in ORM is used.

    No built-in ORM - SQLAIchemy is used.

    4

    Design

    Batteries-included - more industry-oriented and therefore comes with built-in features.

    Minimalistic design - provides only the scaffolding for a web app.

    5

    Working Style

    Monolithic - The developer needs to follow certain rules to run the Django Project

    Diversified - developers can use many web tools.

    Future and Conclusions

    Django was initially developed to manage one-side, synchronous and request-response interactions based on HTTP protocol. There was no support for non-HTTP protocols like websockets.
    Django 3.1, which just released a few weeks earlier, has asynchronous protocol support. This version is backward compatible and which ensures a shining future of Django in both sync and async protocols!
